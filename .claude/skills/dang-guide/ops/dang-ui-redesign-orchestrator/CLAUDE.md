# dang-ui-redesign-orchestrator

## Purpose
- Standardize how DangApp turns UI research into route-level redesign decisions and implementation order.

## Structure
- `SKILL.md`: trigger conditions, fixed inputs, orchestration modes, and output contract.
- `references/`: reusable reference clusters and the screen mapping template.

## Responsibilities
1. Normalize deep research into DangApp-specific visual directions.
2. Map references and patterns to actual DangApp routes and shared components.
3. Produce implementation-ready redesign guidance before UI code changes begin.
